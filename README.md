## Установка

```bash
git clone <repo>
cd whitelist-vps-checker
npm install
```

## Использование

### Из исходников

```bash
node cli.js <input.txt> [output.json]
```

### Собранная версия

```bash
npm run build
node dist/vps-checker.js <input.txt> [output.json]
```

| Аргумент | Обязательный | Описание |
|----------|:---:|----------|
| `input.txt` | да | Файл с VLESS-ссылками |
| `output.json` | нет | Файл результата (по умолчанию `result.json`) |

### Пример

```bash
node cli.js vless_universal.txt result.json
```

## Что делает

1. **Парсинг** — извлекает IP из строк вида `vless://UUID@IP:PORT?params#label`
2. **Дедупликация** — оставляет только уникальные IP
3. **Проверка доступности** — TCP connect к каждому IP (порт 80, параллельно до 20 соединений)
4. **Определение провайдера** — batch-запрос к [ip-api.com](http://ip-api.com) (ISP, ASN)
5. **Группировка** — по подсетям /16 и по провайдеру
6. **Вывод** — ASCII-таблица в консоль + JSON-файл

## Формат вывода

### Консоль

ASCII-таблица с секциями:
- **STATS** — общая статистика (всего строк, уникальных IP, живых, мёртвых)
- **PROVIDERS** — список провайдеров с bar-chart
- **SUBNETS** — дерево IP, сгруппированных по /16 подсетям

## Структура проекта

```
├── cli.js                 — точка входа CLI
├── src/
│   ├── parser.js          — извлечение уникальных IP из текста
│   ├── checker.js         — TCP connect проверка доступности
│   ├── resolver.js        — ISP/ASN через ip-api.com (batch)
│   ├── grouper.js         — группировка по подсети и провайдеру
│   ├── formatter.js       — ASCII-форматирование результатов
│   ├── writer.js          — запись JSON
│   └── index.js           — пайплайн (оркестрация)
├── tests/                 — Jest-тесты для каждого модуля
├── dist/                  — собранный бандл (после npm run build)
└── package.json
```

## Скрипты

```bash
npm test          # запуск тестов (Jest, 24 теста)
npm run build     # сборка в dist/vps-checker.js (esbuild, ~7kb)
npm start         # запуск из исходников (node cli.js)
```

## Технологии

- **Node.js** (>=18)
- **Jest** — тестирование
- **esbuild** — сборка в один файл
- **p-limit** — контроль параллелизма
- **net** / **http** — встроенные модули (без внешних HTTP-клиентов)

## Ограничения

- **ip-api.com** бесплатный тариф: 45 req/min, batch до 100 IP за запрос
- TCP таймаут: 3 секунды
- Параллелизм TCP: до 20 одновременных соединений
